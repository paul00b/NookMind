What visual style for the onboarding illustrations?

NookMind uses dark (#0f1117) with amber accents and mode-specific gradients (amber/indigo/teal). Here are 3 directions for the slide visuals.

A

Emoji / Icon Compositions

Large, friendly emoji clusters or Lucide icons (already in the project) arranged into visual scenes. E.g. Slide 1 shows a grid of 🎬📚📺, Slide 2 shows ⭐🔔📊 etc.

Lightweight, loads instantly, consistent across devices. Playful and modern. No external assets needed.

Pros

  • Zero asset weight
  • Works perfectly in dark & light mode
  • Easy to maintain

Cons

  • Can feel generic
  • Less "premium" feel
B

CSS-Illustrated Mockups

Mini-mockups of the actual app screens built in CSS — a tiny phone frame showing the library view, the discover page, a notification card. Users see what they'll actually get.

Shows the real product. Uses the app's own design tokens (amber, dark bg, rounded cards).

Pros

  • Sets accurate expectations
  • Feels polished & branded
  • Reuses existing design system

Cons

  • More complex to build
  • Needs updating if UI changes
C

Abstract Gradient Art + Icons

Full-bleed gradient backgrounds (matching the mode colors: amber, indigo, teal) with floating Lucide icons and bold typography. Each slide has its own color identity.

Vibrant, immersive feel. Each slide has a distinct color that maps to NookMind's media modes (books=amber, movies=indigo, series=teal).

Pros

  • Visually striking
  • Introduces the color language early
  • No external assets, pure CSS

Cons

  • More abstract — less concrete about features
